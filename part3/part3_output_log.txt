Gini impurity: Gini(t) = 1 - sum_i(p_i^2), measures the probability of misclassifying a randomly chosen element if labeled by the class distribution at that node.
Entropy: H(t) = -sum_i(p_i * log2(p_i)), measures the information/disorder at a node in bits. Both criteria typically select similar splits in practice; entropy is marginally more computationally expensive due to the logarithm.

This decorrelates individual trees -- since no single strong feature can dominate every tree's early splits -- so averaging their predictions reduces variance far more than a single deep tree could achieve alone. Grid vs Randomized Search trade-off: GridSearchCV exhaustively evaluates all 27 configurations, guaranteeing the optimum within the specified grid but scaling multiplicatively with each added hyperparameter/value. RandomizedSearchCV instead samples a fixed budget of random configurations from the same space -- for large grids this typically finds a near-optimal configuration in a fraction of the compute, at the cost of no formal optimality guarantee. At this grid's size (27 configs), exhaustive search is still cheap enough to be the better choice; it stops being practical once individual hyperparameters have many more values or more hyperparameters are added.
